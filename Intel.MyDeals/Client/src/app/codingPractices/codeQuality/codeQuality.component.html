My Deals Code Quality Standards

MyDeals is a full stack UI application with complex logic and multiple teams are working on it, it is necessary to integrate code quality tools to follow good coding standards.

  • In that way developers will have proper guidelines and minimum processes to follow.
  • This practice can help to enforce strong coding standards; application will have low security threats and better performance.

Quality Checks

Following the below Quality checks as part of MyDeals Coding Quality Standards

  • Unit Testing(NUnit Test Suite): NUnit is unit test framework for all .Net languages.
  • SonarQube: Tool for static analysis of code for any bugs and reviews.
  • Spectral Scan: Tool to validate security keys in the source.
  • UIPath: Platform that offers comprehensive solutions for UI automation testing, particularly through its UiPath Test Suite.
  • JMeter: Tool for performance testing.
  • Semgrep: Static analysis tool designed for searching code, finding bugs, and enforcing code standards.
  • Dynatrace: Is a Software as a Service offering that focuses on performance and availability monitoring​

On each PR, automated quality checks (Sonar Scan and NUnit Test) will run, and code merging will be blocked unless they pass successfully.

Unit Testing

Unit testing can be done using many different tools and based on the application's infrastructure. My Deals will focus on writing unit test cases in the Middle Tier layers. NUnit is enabled only for files having C# changes.

  • NUnit Test cases must pass in local system before raising PR. On each PR, automated unit tests will be triggered.
  • If Nunit test result is 'PASS' -> No action required can Proceed to next stage.
  • If test result is 'FAIL' -> Single test failure will fail the workflow and code merge will be blocked.
  • Merging will be blocked unless the workflow passes. Each developer raising PR must ensure that it passes successfully.

Refer to Coding Practices -> Coding Tools -> Unit Testing for more information related to Unit Testing

SonarQube

SonarQube is a self-managed, automatic code review tool that systematically helps to deliver Clean Code by integrating in to existing workflow and detects issues in the code to perform continuous code inspections.

  • In MyDeals, for now Sonar is enabled only for Angular changes. SonarLint plugin in VS Code helps to identify the issues in local system before raising the PR.
  • On each PR, a sonar scan will be triggered and merging will be done once sonar checks passed successfully.
  • If sonar workflow fails , the code will not be mergeable, and we must investigate the issue and fix it.
  • Sonar has dashboard from where we can identify the blocker issues.The issue will have recommended solution to fix the bug.
  • For now, the sonar scan workflow will fail only if there is any blocker issue. Low priority vulnerabilities are not considered for now.

UIPath

UIPath is used to automate repetitive business and IT processes by employing Robotic Process Automation (RPA) and AI-powered technologies, allowing businesses to improve efficiency, reduce costs, and free up human employees for higher-value work.

JMeter

Apache JMeter is a Java-based open-source tool designed for load testing and performance measurement of web applications. It has a very user-friendly GUI which makes things easier while scripting and supports various protocols (HTTP, HTTPS, FTP,JDBC etc.).

  • Test scenarios in JMeter are built by configuring essential elements within the test plan: Define thread groups for virtual users and test duration, configure samplers to simulate user requests, add logic controllers for interaction flow, incorporate assertions to validate server responses, introduce timers for realistic delays between actions, parameterize test data using variables from external sources, and iteratively validate and optimize the test plan to accurately simulate user behavior and measure performance metrics.

Semgrep

Semgrep Code is a static application security testing (SAST) tool that detects security vulnerabilities in our code.

  • Semgrep has two methods of functionality: IDE and Scan. For the scan, scans are performed in two ways: full and differential.
    • A full scan is performed on every merge/pull request.
    • A differential scan is performed on branches after every commit.
  • Semgrep Code uses rules, which encapsulate pattern matching logic and data flow analysis, to scan your code for security issues, style violations, bugs, and more.
  • Semgrep AppSec Platform displays Semgrep Code's findings. Additionally, the platform allows you to:
    • Triage findings
    • Send alerts and notifications or create tickets to track findings identified by Semgrep Code
    • Customize how Semgrep Code scans your repositories
    • Manage your users and facilitate team collaboration in remediating security issues
  • Semgrep scans altered files based on schedule that runs at 8am IST on Tuesday and Thursday.

The Semgrep dashboard is an overview of your organization’s security posture based on data aggregated within Semgrep AppSec Platform.

Dynatrace

Dynatrace monitoring is an observability and application performance monitoring (APM) platform that uses artificial intelligence (AI) to provide comprehensive visibility into the entire technology stack, from user experience to infrastructure and code.

  • Monitoring can involve synthetic monitors or agent-based monitors to trace avaialability.
  • Dynatrace enables you to: Use readymade dashboards to monitor your system status in real-time
