Getting Started For Developer

This document help developers to understand MyDeals onboarding process.

How to get Access ?

To access MyDeals application, a developer should raise AGS requests.

  1. Click here to request access from Intel Access Governance system, on click you will be navigated to the home page as shown.
    2.     
  2. Click here to Request Access For Myself, on click you will be directly navigated to the search access page as shown below.



    3.     

  3. Type MYDEALS SOx RO DATABASEACCESS on AD-COR and search, select the checkbox and request for access.

To access MyDeals application, developer should apply below AGS roles.

  1. MYDEALS SOx RO DATABASEACCESS on AD-CORP
  2. Common Architecture Database - RDMS CA Developer
  3. Common Architecture Database - RDMS Prod Read
  4. Common Architecture Database - RDMS CA Developer Role DEV
  5. Intel Cloud Hosting - Intel Cloud Application Platform - Internal - Production
  6. DevTools - - BOS Platform Operation - Project Developer
  7. Common Architecture Database - RDMS Release User DEV

To access MyDeals GITHUB, developer should apply below AGS roles.

  1. 1Source Github User
  2. 1Source Github Collab User on AZAD-CORP
  3. MyDeals Github Developer
Click here for Github SetUp.

Note: Refer AGS Requests section to apply for above roles. Once all the requests are approved it takes approx 24 hours to reflect in the system.

Local Project Setup

After all AGS requests are approved then developer can setup the code in local system.

Click here to install the prerequisites under Local Environment setup.

  1. Click here to open GitHub.

    On click of the above link developer will be nagivated to the github where UI source code can be cloned by copying the URL.

    2.     

  2. Open Visual Studio IDE and click on clone a repository, paste the URL and click on clone as shown below.

    3.     

  3. Once the cloning is successfully completed the developer can see the .sln files, select Intel.Mydeals.sln

    4.     

  4. Right click on Intel.Mydeals and select Set as StartUp Project and open the terminal in client folder then run the command npm run build. After successful build, run the application from visual studio.

Onboarding process for DB

  1. Type Common Architecture Database - RDMS Release User DEV and search, select the checkbox and request for access.


  2. Download and install the SSMS (Sql Server Management Studio).
  3. Open the SSMS, select the required fields and connect with database.
  4. Dev Sever connection string: FM7DFROAPP01.amr.corp.intel.com
    Note: For other pre-prod and prod server connection strings, please reach out to your respective Tech Leads.
    5.   
  5. Click here to open GitHub.
    On click of the above link developer will be nagivated to the github where DB source code can be cloned by copying the URL.
  6. Once the cloning is successfully completed the developer can see the folder view.
