SSIS Batch Job Process

  1. How to access SSIS screen?

    • To access Batch Job constant screen, empId should be added in SSIS_CNST_EMP_ID attribute
      To verify access, GoTo Admin -> Dev Tools -> Monitoring -> constants
    • For accessing the batch job constant screen, GoTo Admin -> Dev Tools -> Monitoring -> Batch Job Constants
    • If you have access the batch job constant screen will load or otherwise, it will throw the error User does not have access to the screen. Press OK to redirect to Dashboard in alert pop-up.
    • Batch job constant screen with access

  2. Types of batch jobs in MyDeals

    • Autosys

      • Purpose: Autosys is a enterprise job scheduling tool used for managing and automating the execution of batch jobs across the various platforms
      • Features: It provides capabilities for scheduling, monitoring and managing jobs and workflows, handling complex dependencies and condition, and integrating with different systems and applications.

    • SQL Agent Job

      • Purpose: SQL Server Agent is a component of Microsoft SQL Server used specifically for automationg tasks and managing job execution related to SQL Server databases.
      • Features: It offer job scheduling, aleting, and automation capabilities directly within SQL Server. It can handle tasks such as running Transact-SQL script, SSIS packages, and managing backups.

  4. Steps to adhoc run a batch jobs (Manually via batch job constant screen)

    • Visit the batch job constant screen, GoTo Admin -> Dev Tools -> Monitoring -> Batch Job Constants
    • When you select the edit button for a specific batch task, the batch job details are displayed. If you see that the toggle button for adhoc runs is OFF.
    • To execute batch job constants manually, Turn ON the adhoc run toggle button and Turn ON the specific step adhoc run toggle button as well.
    • On successful save, the selected batch job will have the adhoc toggle button Turned ON and the job will be enabled for adhoc execution (which will be picked by a runner at fixed interval).
      NOTE: Once the batch job starts executing the status will be changed to running and once it's execution complete the status will reflect as completed/failed based on pass/failure scenario and the adhoc button will be turned OFF.
    • After that, the successful message "Batch Job Constants Data Updated Successfully" will appear.
